1. Problem Statement 
Manual grading of descriptive answers is a labor-intensive, time-consuming process that is often 
prone to subjectivity and inconsistency across different evaluators. As class sizes increase, 
instructors face significant challenges in maintaining grading quality and speed. There is a 
critical need for an intelligent system that can assist educators by evaluating student answers 
objectively, ensuring consistency based on predefined rubrics and model solutions. 
2. Proposed Solution 
We propose an Automated Answer Evaluation System utilizing Large Language Models 
(LLMs) and Retrieval Augmented Generation (RAG). This system aims to streamline the 
grading process by automatically assessing student responses against teacher-provided criteria. 
The workflow involves: 
 Input: Teachers upload exam questions and detailed grading rubrics. 
 Processing: The system generates a model reference solution and stores it alongside the rubric in 
a vector database. 
 Evaluation: When a student submits an answer, the system retrieves the relevant reference 
material using RAG and compares the semantic meaning of the student's answer against the 
model solution. 
 Output: The system generates a score and feedback based on alignment with the rubric. 
 
 
 
3. Functional Requirements 
3.1 Teacher Module 
 Exam Management: Interface to upload exam question papers. 
 Rubric Management: Capability to upload and define grading rubrics (criteria and marks 
distribution). 
 Solution Generation: Trigger the LLM to generate model/reference solutions for the uploaded 
questions. 
 Vectorization: Functionality to convert and store all reference documents as vector embeddings. 
 Result Review: Dashboard to view evaluated results, scores, and system-generated feedback. 
3.2 Student Module 
 Submission Portal: Interface for students to upload their descriptive answers for a specific exam 
ID. 
 Result Viewing: (Optional) Interface to view awarded marks and feedback after the grading 
process is complete. 
3.3 AI Grading Module 
 Embedding Generation: Convert exams, rubrics, and reference solutions into high-dimensional 
vector embeddings. 
 Storage: Store embeddings efficiently in a vector database. 
 Retrieval (RAG): Retrieve relevant reference content contextually matching the student's 
answer. 
 Semantic Evaluation: Compare student answers with reference solutions using semantic 
similarity measures. 
 Scoring Engine: Assign scores strictly based on the rubric alignment and similarity thresholds. 
3.4 Data Storage 
 Vector Database: Store document chunks and embeddings. 
 Relational/Metadata Store: Maintain metadata including Exam IDs, Question Numbers, 
Student IDs, and Max Marks. 
4. Non-Functional Requirements 
 Accuracy: The output must align with the criteria defined in the rubric to ensure fairness. 
 Consistency: The system must be deterministic; the same answer should receive similar scores 
across multiple evaluation runs. 
 Scalability: The architecture must support concurrent uploads for multiple students and exams. 
 Security: Role-based access control (RBAC) to ensuring only authorized users (Teachers) can 
upload rubrics or view all student grades. 
 Usability: The interface must be intuitive, allowing for simple document uploads and clear 
results.

6. Technology Stack 
 LLM: GPT-4o / Llama 3 (via API or local hosting) 
 Vector Database: Pinecone / ChromaDB / FAISS 
 Backend Framework: Python (FastAPI or Flask) 
 Embeddings: OpenAI Embeddings / HuggingFace Sentence Transformers 
 Frontend: React / Streamlit (for prototype interface)
